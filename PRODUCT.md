# Odyssey.md — 产品定义 v0.6 (MVP 视觉完成)

> Your life is an Odyssey.

## 一句话

一个 Obsidian 插件,把你的人生事件同时呈现在丝滑的日历和美丽的地图上,并能以电影感的方式回放。

## 统一的 Entry

一个 Entry = 一段时间(必须) + 一个 location(可选)。时间可长可短,地点填了就上地图,不填就只在日历上。

## Schema

| 字段       | 必填 | 说明                         |
| ---------- | ---- | ---------------------------- |
| date       | ✅   | 事件日期 (ISO: YYYY-MM-DD)   |
| start_time |      | 起始时间 (HH:MM)             |
| end_date   |      | 结束日期(不填 = 当天)       |
| end_time   |      | 结束时间                     |
| location   |      | 单个地点(MVP 阶段为文本)   |

## 日历 vs 地图:承载不同的子集

- **日历**:承载所有 Entry(时间是唯一必须字段)
- **地图**:只承载有 location 的 Entry
- **Replay**:只回放有 location 的 Entry(没地点就没法在地图上画)

这个不对称是**刻意的**:不是每件事都落到一个地点(读完一本书、一个 deadline、一次内心的波动),强制填 location 会把产品变成 friction 工具。

## 三种渲染形态(插件自动判断,用户无感)

- 只有 `date` → 全天卡片
- `date` + 时间(同一天)→ 时间块
- `date` + `end_date`(跨天)→ 跨天长条

## 四大核心能力

1. 📝 Entry 创建:一套表单,字段按需填
2. 📅 日历视图:丝滑程度 ≥ Notion Calendar
3. 🗺️ 地图视图:美观程度 ≥ Polarsteps
4. 🎬 Replay 模式:选时间范围,按播放,重温人生

## Replay 模式(灵魂功能)

- 入口:地图视图上的一个按钮
- 操作:选起止日期 → 播放
- 镜头:跟飞(跟随当前 Entry 移动)
- 点迹:累积留痕(历史点亮着不灭)
- 可调速,显示当前时间和 Entry 标题
- 情绪定位:旅行重温,不是数据查询

## 视图联动

悬停高亮 / 点击跳转 / 范围筛选 / 并排同步

## 不做(留给未来)

- 多地点轨迹(单 Entry 多 location) → v0.5+
- 子事件嵌套 → v0.5+
- 分类 / 标签 / 搜索 / 统计等传统 PKM 功能 → 视情况

## 核心哲学

- 隐形复杂度:用户只管记录,插件处理一切
- 时间跨度是"旅程感"的本质,空间回放是高潮
- 视觉即产品,仪式感即价值

## Date Range 的承诺

从 MVP 第一天就支持时间段 Entry,不重蹈 Anytype 四年不加此功能的覆辙。
